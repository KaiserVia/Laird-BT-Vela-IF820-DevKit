## Pins – UART-Schnittstelle

Quelle: DS50002519G, Table 1-1

| Pin-Nr. | Name | Richtung | Funktion |
|---|---|---|---|
| **22** | **RXD** | Input (in das Modul) | UART Dateneingang |
| **23** | **TXD** | Output (vom Modul) | UART Datenausgang |
| **15** | **P1_7 / CTS** | Input (in das Modul) | UART CTS – stoppt Modul wenn HIGH |
| **17** | **P0_0 / RTS** | Output (vom Modul) | UART RTS – Modul zieht LOW wenn empfangsbereit |

Aus DS50002519G §2.2:
> "The RTS pin can only be assigned to P0_0 and the CTS pin can only be assigned to P1_7."

Die GPIO-Belegung der UART-Pins im SX-Befehlssystem (DS50002506C, Table 2-7):

| SX-Pin-Nr. | Modul-Pin | Standard-Funktion |
|---|---|---|
| 00 | P0_0 | UART RTS (Aenderung vermeiden) |
| 02 | P1_7 | UART CTS (Aenderung vermeiden) |

## Baudrate einstellen – `SU` Befehl

Quelle: DS50002506C, Table 2-6

Command `SU` sets the UART baud rate. Table 2-6 shows the input parameters and their corresponding baud rates.

Syntax: `SU,<H8>` + `\r` → Antwort `AOK`

| Index (hex) | Baudrate |
|---|---|
| `03` | **115200** ← Default |
| `04` | 57600 |
| `05` | 38400 |
| `06` | 28800 |
| `07` | 19200 |
| `08` | 14400 |
| `09` | 9600 |
| `0A` | 4800 |
| `0B` | 2400 |
| **`10`** | **307200** ← im Projekt genutzt |
| `11` | 1000000 (1 Mbit/s) |

Die neue Baudrate wird erst nach einem Reboot (`R,1`) wirksam.

**Im Projekt:** `SU,10\r` → 307200 bps. Danach `R,1\r`, MCU-UART auf 307200 umstellen.

## Hardware-Flowcontrol – `SQ` Befehl

Quelle: DS50002506C, DS50002519G §2.2

Flowcontrol ist per Default **aktiviert** (SQ-Bit `0x1000` gesetzt).

```
SQ,1000   # Flowcontrol aktivieren (Default)
SQ,0000   # Flowcontrol deaktivieren
```

Funktionsweise:
- **CTS (Pin 15, Input):** Wenn CTS HIGH → Modul pausiert Sendung. Wenn CTS LOW → Modul sendet.
- **RTS (Pin 17, Output):** Modul zieht RTS LOW wenn es Daten empfangen kann. RTS HIGH wenn Puffer voll.

Fuer den LPC1766: Hardware-Flowcontrol muss in `Init_UART1()` mit `U1->MCR |= (1<<7)|(1<<6)` aktiviert sein.

## Command Mode – Einstieg und Verlassen

Quelle: DS50002506C

### Einsteigen
- Sende `$$$` (drei Dollarzeichen, **ohne** CR)
- Modul antwortet mit `CMD>` wenn bereit
- **Kein Guard Time** – funktioniert jederzeit im Data Mode
- Escape-Zeichen aenderbar mit `S$,<char>` (Default: `$`)

### Verlassen
- Sende `---\r` → Modul antwortet mit `END`, kehrt in Data Mode zurueck

### Sonstiges
- Alle Befehle mit `\r` (0x0D) abschliessen
- Antworten: `AOK` = Erfolg, `ERR<n>` = Fehler
- `CMD>` Prompt vor jedem Befehl (deaktivierbar mit `SQ,4000`)
- `<ESC>` (0x1B) im Command Mode: loescht laufende Eingabe

## Statusmeldungen – `SO` Befehl

Quelle: DS50002506C, Table 2-4

Syntax: `SO,<prefix>,<suffix>` + `\r`  
Default: `%,%` → Meldungen erscheinen als `%CONNECT%`, `%DISCONN%` usw.

**Alle Meldungen deaktivieren:** `SO,,\r` (leere Parameter) ← im Projekt genutzt

| Meldung | Ereignis |
|---|---|
| `CONNECT` | Verbindung zu Peer hergestellt |
| `DISCONN` | Verbindung getrennt |
| `REBOOT` | Modul startet neu |
| `BONDED` | Neues Bonding gespeichert |
| `RFCOMM_OPEN` | SPP-Stream aktiviert |
| `RFCOMM_CLOSE` | SPP-Stream deaktiviert |
| `LSTREAM_OPEN` | BLE UART Transparent Stream offen |
| `AUTHENTICATED` | BT Classic Authentifizierung erfolgreich |
| `AUTH_FAIL` | BT Classic Authentifizierung fehlgeschlagen |
| `ERR_CONN` | Verbindungsversuch fehlgeschlagen |
| `FACTORY_RESET` | Factory Reset via Hardware-Pin |
| `SECURED` | BT Classic Link verschluesselt |

## Authentifizierung – `SA` Befehl

Quelle: DS50002506C

Syntax: `SA,<1-4>` + `\r`  
Default: `SA,2`

| Wert | Modus | Beschreibung |
|---|---|---|
| `SA,1` | SSP Confirm | 6-stellige PIN auf beiden Seiten angezeigt, Benutzer bestaetigt |
| `SA,2` | SSP Just Works | Kein PIN noetig ← **Default** |
| `SA,3` | SSP Input | PIN wird auf Remote-Seite angezeigt, lokal eingeben |
| `SA,4` | Legacy PIN | BT Classic 2.0, 4-stellige PIN ueber `SP`-Befehl ← im Projekt genutzt |

**Im Projekt:** `SA,2\r` → Just Works, kein PIN.  
Mit PIN: `SA,4\r`, dann `SP,<6-stellige-PIN>\r`

## Initialisierungsablauf im Projekt (btio.c)

Quelle: `btio.c: init_bluetooth()`, `btio.h`

Das Modul wird bei Erstinbetriebnahme vollstaendig neu konfiguriert:

```
Init_BT_ch(115200)          # UART1 + MUX auf BT, 115200 Baud (Default nach Factory Reset)

# Falls kein CMD> empfangen: mit 307200 versuchen (evtl. schon konfiguriert)
$$$                         # -> CMD> Prompt
SF,1\r                      # Factory Reset -> AOK -> Reboot laeuft an
R,1\r                       # Reboot
Init_BT_ch(115200)          # zurueck auf 115200

# Warte auf "%REBOOT" Meldung vom Modul (bis 13s)
$$$                         # -> CMD> Prompt nach Reboot

SN,VIASIS_<serno>\r         # Device Name setzen -> AOK
SA,2\r                      # Authentifizierung: Just Works -> AOK
SS,SPP\r                    # BT Classic Service Name: SPP -> AOK
SY,4\r                      # Max. Sendeleistung -> AOK
SP,<6-stellig>\r            # PIN (letzte 6 Digits der Seriennummer) -> AOK
SG,0\r                      # Dual Mode (BLE + Classic) -> AOK
SU,10\r                     # Baudrate auf 307200 -> AOK
SO,,\r                      # Status-Meldungen deaktivieren -> AOK

R,1\r                       # Reboot ohne Warten auf Bestaetigung
Init_BT_ch(307200)          # Host jetzt auf 307200 Baud
# Warte 1s

$$$                         # -> CMD> Prompt (auf 307200 Baud)
---\r                       # Zurueck in Data Mode -> END

# fp.btmodem = Roving, interfaces |= BT_LINK
```

**Wichtig:** `SU,10` (307200) wird zuletzt gesetzt. Der Reboot danach aktiviert die neue Baudrate. Der LPC1766 muss nach `R,1` den UART sofort auf 307200 umstellen (`Init_BT_ch(307200)`), bevor er wieder kommuniziert.

## Identifikation im Projekt

Quelle: `btio.h`

```c
#define Roving   46          // Microchip RN4678APL
#define T_rn4678 " Micro"   // Erkennungsstring: Modul gibt " Micro..." als Hersteller zurueck
```

Das Modul wird erkannt, wenn auf `$$$` mit `CMD>` geantwortet wird und kein Laird vorhanden ist. Der Erkennungsstring `" Micro"` taucht in der Firmenantwort des RN4678 auf.

## Betrieb nach Konfiguration

- **Baudrate:** 307200 bps
- **Format:** 8N1
- **Flowcontrol:** RTS/CTS aktiv
- **Command Mode:** `$$$` (kein CR), Antwort `CMD>`
- **Data Mode:** `---\r`, Antwort `END`
- **Statusmeldungen:** deaktiviert (`SO,,`)
- **Profil:** SPP (Serial Port Profile) fuer BT Classic
